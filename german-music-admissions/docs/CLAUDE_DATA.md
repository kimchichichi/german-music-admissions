# CLAUDE_DATA.md — universities.json 데이터 규칙

> universities.json을 읽거나 수정하는 작업 시 이 파일을 읽어라.
> 마스터 규칙(CLAUDE.md)도 반드시 읽어야 한다.

---

## 이 파일의 역할

`universities.json`은 이 프로젝트의 **단일 진실 소스(Single Source of Truth)**.
24개 학교의 모든 입시 정보가 여기서 관리된다.

---

## null 규칙 (가장 중요)

```
null = "아직 공식 확인 안 됨"
"없음" = "해당 사항 없음" (명시적으로 확인된 경우)
```

| 표현 | 의미 | 예시 |
|------|------|------|
| `null` | 미확인, 아직 모름 | 마감일을 아직 못 찾았을 때 |
| `"없음"` | 확인했더니 없음 | 여름학기 없는 학교 |
| `"미확인"` | data_status에서만 사용 | 전체 학교 상태 표시 |

---

## data_status 3단계

```
"확인됨"    → 지은님이 공식 원문 직접 확인 완료
"부분확인"  → 일부만 확인, null 필드 남아있음
"미확인"    → 공식 확인 전혀 안 됨
```

Claude는 절대로 "미확인" → "확인됨"으로 임의 승격하지 말 것.
반드시 지은님이 공식 페이지/PDF 확인 후 직접 변경.

## 핵심 필드 설명

```json
{
  "lang_timing": "지원전" | "조건부" | "등록전" | "없음" | null
}
```

- `"지원전"` — 지원서 제출 전에 증명서 첨부 필수
- `"조건부"` — 언어증명서 없이 입학 허용, 기한 내 미제출 시 자격 소멸
- `"등록전"` — 합격 후 등록(Immatrikulation) 전까지 제출
- `"없음"` — 언어 조건 자체가 없음

```json
{
  "portal_type": "muvac" | "online" | "post"
}
```

- `"muvac"` — muvac.com 플랫폼 사용
- `"online"` — 학교 자체 온라인 포털
- `"post"` — 온라인 + 우편 병행 필수

**미확인 6개:** 에센(8) · 만하임(16) · 뮌헨(17) · 로스토크(19) · + 확인 필요

## Claude가 할 수 있는 것 vs 할 수 없는 것

| 할 수 있음 | 할 수 없음 |
|-----------|-----------|
| null 필드 목록 뽑아주기 | null을 추측값으로 채우기 |
| 공식 URL 구조 안내 | 훈련 지식으로 데이터 확정 |
| 지은님이 확인한 값 입력 대신 타이핑 | data_status 임의 변경 |
| JSON 문법 오류 수정 | 학교 데이터 재생성 |
